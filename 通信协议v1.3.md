## 2. 心跳机制
### 2.1 心跳请求 (device → server)
```json
{
    "cmd": "heartBeat",
    "signal": 23, // 4g信号强度,对应5格信号[0~5, 6~11, 12~16, 17~21, 22~32]
    "gps": {
        "latitude": "2306.3446N",
        "longitude": "11302.3336E",
        "state": 1 // 值为1时，gps信息有效
    },
    "stateList": [ // 多仓门状态列表
        {
            "smoke": 0,      // 烟雾报警状态， 0为正常， 1检测到烟雾
            "capacity": 2,   // 容量[0可投递, 1红外遮挡, 2超重, 3红外遮挡后-投递超重]
            "weigh": 62.2,   // 当前货物重量
            "doorStatus": 0,  // 投递门业务状态[0空闲, 1投递中]
            "rodStatus": 100, // 投递门实际状态[100开门成功，101开门中 102~199 开门失败]  [200关门成功, 201关门中, 202~299为关门失败]
            "lockStatus": 0,  // 清运门锁状态[关锁状态=0, 开锁状态=1]
            "irState": 1,     // 红外状态[0通过 1阻挡]
            "cabinId": "20250524143325016956"// 仓门ID
        },
        {
            "smoke": 0,
            "capacity": 0,
            "weigh": 0,
            "doorStatus": 0,
            "rodStatus": 100,
            "lockStatus": 1,
            "irState": 0,   
            "cabinId": ""
        }
    ],
    "timestamp": "1748325851583"
}
```

## 3. 开门流程

```json
{
  "cmd": "openDoor",
  "openType": 1, // 开门类型[1投递门， 2清运门]

  "cabinId": "20250604143922487613", // 仓门ID
  "transId": "delivery17525489033965821", // 事务ID
  "phoneNumber": "10086",
}
```
### 3.2 开门响应 (device → server)
```json
{
  "cmd": "openDoor",
  "retCode": 0, // 0开门成功，非0失败
  "transId": "delivery17525489033965821",
  "cabinId": "20250604143922487613",
  "phoneNumber": "10086",

  "curWeight": 0.26048983740995747,
  "timestamp": "1752548903786"
}
```

### 3.3 设备在执行开门操作中检测到错误后取消开门 (device → server) (停用)
```json

  "phoneNumber": "10086",
```

## 二:  HTTP通信接口
## 1. 上传设备照片 
- **请求地址**：uploadPhotoURL
- **协议**：HTTP/HTTPS  
- **方法**：POST  
- **Content-Type**：`multipart/form-data`

### 字符串参数（Form-Data）

| 参数名      | 类型   | 必填 | 说明                          |
|-------------|--------|------|-------------------------------|
| sn          | string | ✓    | 设备序列号 (长度8-32位)        |
| iccid       | string | ✓    | SIM卡标识 (长度20位)           |
| transId     | string | ✓    | 事务ID (格式: UUID或时间戳)    |
| cabinId     | string | ✓    | 仓门ID  |
| photoType   | string | ✓    | 照片类型：<br>`0`-内照-开门<br>`1`-内照-关门<br>`2`-外照-开门<br>`3`-外照-关门<br>`4`-内照-运维<br>`5`-外照-运维

### 文件参数

| 参数名 | 类型   | 必填 | 说明                     |
|--------|--------|------|--------------------------|
| file   | binary | ✓    | 上传的文件（图片/日志等）|


## 请求示例
```bash
curl -X POST "https://api.example.com/api/v1/upload/file" \
  -F "sn=DEV202307001" \
  -F "iccid=89860620050035548765" \
  -F "transId=ID_1752548903786" \
  -F "cabinId=20250604143922487613" \
  -F "photoType=1" \
  -F "file=@/path/to/photo.jpg

  ```
